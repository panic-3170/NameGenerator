# 网名生成器 - 实现方案

## 一、产品定位

输入一段简短描述（如「跨境电商」），输出一组（10-20 个）**音韵优美、好记、有来历/寓意**的网名。

**核心卖点**：不是「字典随机拼字」，而是「好听 + 有意境 + 可当作品牌」。

## 二、精准音韵美感如何实现

### 1. 音韵评分体系（核心）

给每个候选名打分，0-100 分：

| 维度 | 权重 | 评分逻辑 |
|------|------|----------|
| 声调起伏 | 25% | 4 字名：二四声优先（上声 + 去声）；避免 4 个同声调 |
| 押韵 | 20% | 末字落在 i/ang/u/an 等响亮韵母 |
| 平仄 | 15% | 双数位以平声为主，末字以仄声收尾 |
| 字形 | 10% | 左右/上下结构优先，避免堆叠 |
| 寓意 | 20% | 引用诗词/典故/自然意象，加分 |
| 易读性 | 10% | 首字母组合是否朗朗上口 |

### 2. 音韵数据库结构

```typescript
type NameChar = {
  char: string;
  pinyin: string;
  tone: 1|2|3|4;       // 声调
  rhyme: string;        // 韵母
  meaning: string;
  source?: string;      // 典故出处
  category: string[];   // 意象分类
};
```

**数据量**：
- 精选 2000+ 常用美字
- 预置 100+ 行业主题词库（电商/科技/文艺/二次元…）
- 预置 500+ 已验证好听的 2-4 字名字模板

### 3. 本地生成算法（4 步流水线）

```
1. 关键词提取
   「跨境电商」→ [跨, 境, 电商, 商业, 通]
2. 主题匹配
   匹配「商业/通达/山水/航路」词库
3. 模板组合
   前缀库 × 中缀库 × 后缀库 → 200+ 候选
4. 音韵打分排序
   取 Top 10-20
```

## 三、本地 + API 增强模式

### 模式 A：纯本地（默认，免费）
- 上文 4 步流水线
- 响应 < 100ms
- 离线可用

### 模式 B：API 增强（用户可选开启）
- 拿本地 Top 5 候选作为「种子」传给大模型
- Prompt：「基于以下名字的意境，生成 10 个类似风格但更有新意的名字：[种子列表]」
- 模型返回后再次过音韵评分
- 显示「AI 增强」标签

### 成本控制
- 用户自备 API Key（OpenAI/Claude/Moonshot）
- 本地存储 Key（localStorage）
- 每次生成消耗 ~500 tokens

## 四、页面结构

只有一个核心页面 + 设置弹窗：

**主页（Hero + 输入 + 结果）**
- Hero：标语 + 输入框
- 调节器：字数（2/3/4）、风格（文艺/豪迈/清新/二次元）、数量（10/20）
- 描述输入：「我希望的名字气质是：___」
- 生成按钮
- 结果卡片列表：每个名字 = 大字 + 拼音 + 寓意 + 一键复制
- 「AI 增强」开关 + API Key 输入（折叠）

## 七、为什么这样做能打赢收费软件

| 收费软件做法 | 我们的做法 |
|-------------|-----------|
| 字典随机拼字，名字生硬 | 音韵打分 + 主题词库 |
| 名字千篇一律 | 500+ 模板组合 + 调参空间 |
| 不告诉用户含义 | 每个名字带「寓意 + 出处」 |
| 必须付费解锁 AI | AI 增强可选，用户自带 Key |

## 八、MVP 范围（v0.1）

只做中文 2-4 字名 + 音韵评分 + 主题词库。英文/AI 增强/平台可用性放 v0.2+。
